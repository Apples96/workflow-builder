# Cell-Based Workflow Architecture Specification

## Overview

This document specifies the transformation from a monolithic workflow execution model to a cell-based, step-by-step execution model. Each workflow will be composed of discrete "cells" that are generated and executed sequentially, with results displayed progressively to the user.

---

## 1. Current Architecture (Monolithic)

### Current Flow
```
User Description → [Single Claude Call] → Complete Python Code → [Single exec()] → Final Result
```

### Current Problems
1. **No visibility during execution**: User waits for entire workflow to complete
2. **All-or-nothing failure**: If step 3 of 5 fails, no partial results visible
3. **No step-by-step debugging**: Cannot inspect intermediate outputs
4. **Progress is retrospective**: SSE streams progress AFTER execution completes
5. **Monolithic code blob**: Single `execute_workflow()` function handles everything

---

## 2. Target Architecture (Cell-Based)

### Target Flow
```
User Description
    → [Planning Call] → Workflow Plan (list of steps)
    → [Generate Cell 1] → [Execute Cell 1] → [Display Output 1] → Stream to UI
    → [Generate Cell 2] → [Execute Cell 2] → [Display Output 2] → Stream to UI
    → ...
    → [Generate Cell N] → [Execute Cell N] → [Display Output N] → Stream to UI
    → Final Aggregated Result
```

### Benefits
1. **Real-time visibility**: Each cell's output displayed as it completes
2. **Partial results on failure**: Earlier cells' outputs preserved
3. **Step-by-step debugging**: Inspect each cell's input/output
4. **True streaming**: Results stream as they're produced
5. **Modular code**: Each cell is a discrete, testable unit

## 3. Data Model Changes

### 3.1 New Models (add to `api/workflow/models.py`)

```python
class CellStatus(str, Enum):
    """Status of an individual workflow cell"""
    PENDING = "pending"          # Not yet generated or executed
    GENERATING = "generating"    # Code being generated by Claude
    READY = "ready"              # Code generated, ready to execute
    EXECUTING = "executing"      # Currently running
    COMPLETED = "completed"      # Finished successfully
    FAILED = "failed"            # Execution failed
    SKIPPED = "skipped"          # Skipped due to earlier failure

@dataclass
class WorkflowCell:
    """
    Represents a single executable cell within a workflow.

    Each cell is a discrete step that:
    - Has its own generated code
    - Receives inputs from previous cells via execution_context
    - Produces outputs that can be consumed by subsequent cells
    - Can be executed independently once its dependencies are met
    """
    id: str = field(default_factory=lambda: str(uuid.uuid4()))
    workflow_id: str = ""
    step_number: int = 0

    # Description and planning
    name: str = ""                          # e.g., "Document Search"
    description: str = ""                   # What this cell does

    # Dependencies
    depends_on: List[str] = field(default_factory=list)  # Cell IDs this depends on
    inputs_required: List[str] = field(default_factory=list)  # Variable names needed
    outputs_produced: List[str] = field(default_factory=list)  # Variable names produced

    # Code generation
    generated_code: Optional[str] = None    # The Python code for this cell
    status: CellStatus = CellStatus.PENDING

    # Execution results
    output: Optional[str] = None            # Human-readable output
    output_variables: Optional[Dict[str, Any]] = None  # Variables produced
    error: Optional[str] = None
    execution_time: Optional[float] = None

    # Timestamps
    created_at: datetime = field(default_factory=datetime.utcnow)
    executed_at: Optional[datetime] = None

@dataclass
class WorkflowPlan:
    """
    High-level plan for a workflow, produced during planning phase.

    Contains the sequence of cells to be generated and executed,
    along with their dependencies and data flow.
    """
    id: str = field(default_factory=lambda: str(uuid.uuid4()))
    workflow_id: str = ""
    description: str = ""                   # Original user description

    # Planning output
    cells: List[WorkflowCell] = field(default_factory=list)
    shared_context_schema: Dict[str, str] = field(default_factory=dict)  # Variable name -> type

    # Status
    status: str = "created"                 # created, planning, ready, executing, completed, failed
    current_cell_index: int = 0

    created_at: datetime = field(default_factory=datetime.utcnow)
```

## 5. Executor Changes

### 5.1 New File: `api/workflow/cell_executor.py`

```python
class CellExecutor:
    """
    Executes workflow cells one at a time with state passing.

    Key responsibilities:
    1. Execute cells in dependency order
    2. Pass outputs from one cell to the next via execution_context
    3. Stream progress/results via SSE as each cell completes
    4. Handle failures gracefully (preserve partial results)
    """

    def __init__(self):
        self.max_cell_execution_time = 120  # 2 minutes per cell
        self.execution_contexts: Dict[str, Dict[str, Any]] = {}  # execution_id -> context

    async def execute_workflow_stepwise(
        self,
        workflow: Workflow,
        user_input: str,
        attached_file_ids: Optional[List[int]] = None
    ) -> AsyncGenerator[Dict[str, Any], None]:
        """
        Execute workflow cell by cell, yielding events as they occur.

        Yields events:
        - {"type": "workflow_start", "total_cells": N}
        - {"type": "cell_generating", "cell_id": "...", "cell_name": "..."}
        - {"type": "cell_ready", "cell_id": "...", "code_preview": "..."}
        - {"type": "cell_executing", "cell_id": "...", "cell_name": "..."}
        - {"type": "cell_completed", "cell_id": "...", "output": "...", "variables": {...}}
        - {"type": "cell_failed", "cell_id": "...", "error": "..."}
        - {"type": "workflow_completed", "final_result": "..."}
        - {"type": "workflow_failed", "error": "...", "completed_cells": N}
        """
        execution_context = {
            "user_input": user_input,
            "attached_file_ids": attached_file_ids or []
        }

        yield {"type": "workflow_start", "total_cells": len(workflow.cells)}

        for cell in workflow.cells:
            try:
                # Generate code for this cell if not already generated
                if cell.status == CellStatus.PENDING:
                    yield {"type": "cell_generating", "cell_id": cell.id, "cell_name": cell.name}
                    await self._generate_cell_code(cell, execution_context)
                    yield {"type": "cell_ready", "cell_id": cell.id, "code_preview": cell.generated_code[:200]}

                # Execute the cell
                yield {"type": "cell_executing", "cell_id": cell.id, "cell_name": cell.name}
                cell_result = await self._execute_cell(cell, execution_context)

                # Update context with outputs
                execution_context.update(cell_result["variables"])

                yield {
                    "type": "cell_completed",
                    "cell_id": cell.id,
                    "cell_name": cell.name,
                    "output": cell_result["output"],
                    "variables": list(cell_result["variables"].keys()),
                    "execution_time": cell_result["execution_time"]
                }

            except Exception as e:
                yield {
                    "type": "cell_failed",
                    "cell_id": cell.id,
                    "cell_name": cell.name,
                    "error": str(e)
                }
                yield {
                    "type": "workflow_failed",
                    "error": f"Cell '{cell.name}' failed: {str(e)}",
                    "completed_cells": cell.step_number - 1
                }
                return

        # All cells completed
        final_result = execution_context.get("final_result", "Workflow completed")
        yield {"type": "workflow_completed", "final_result": final_result}

    async def _generate_cell_code(self, cell: WorkflowCell, context: Dict[str, Any]) -> None:
        """Generate code for a single cell using Claude"""
        pass

    async def _execute_cell(self, cell: WorkflowCell, context: Dict[str, Any]) -> Dict[str, Any]:
        """Execute a single cell's code with the given context"""
        pass
```

## 6. API Endpoint Changes

### 6.1 New Endpoint: Stream Execution (`api/main.py`)

```python
@api_router.post("/workflows/{workflow_id}/execute-stream", tags=["Execution"])
async def execute_workflow_stream(workflow_id: str, request: WorkflowExecuteRequest):
    """
    Execute a workflow with real-time streaming of cell results.

    This endpoint executes the workflow cell-by-cell, streaming events
    via Server-Sent Events as each cell generates, executes, and completes.

    Event Types:
    - workflow_start: Workflow execution beginning
    - cell_generating: Claude is generating code for a cell
    - cell_ready: Cell code generated successfully
    - cell_executing: Cell is now running
    - cell_completed: Cell finished with output
    - cell_failed: Cell execution failed
    - workflow_completed: All cells finished successfully
    - workflow_failed: Workflow stopped due to cell failure

    Returns:
        StreamingResponse: SSE stream of execution events
    """
    async def event_generator():
        workflow = workflow_executor.get_workflow(workflow_id)
        if not workflow:
            yield f"data: {json.dumps({'type': 'error', 'message': 'Workflow not found'})}\n\n"
            return

        cell_executor = CellExecutor()
        async for event in cell_executor.execute_workflow_stepwise(
            workflow,
            request.user_input,
            request.attached_file_ids
        ):
            yield f"data: {json.dumps(event)}\n\n"

    return StreamingResponse(
        event_generator(),
        media_type="text/event-stream",
        headers={
            "Cache-Control": "no-cache",
            "Connection": "keep-alive",
            "Access-Control-Allow-Origin": "*"
        }
    )
```

### 6.2 Updated Create Workflow Endpoint

```python
@api_router.post("/workflows", response_model=WorkflowResponse, tags=["Workflows"])
async def create_workflow(request: WorkflowCreateRequest):
    """
    Create a new workflow with cell-based planning.

    This endpoint now:
    1. Creates a workflow plan (list of cells)
    2. Does NOT generate code yet (that happens during execution)
    3. Returns the plan for user review
    """
    # Validate API keys
    validate_anthropic_api_key()

    try:
        # Create the workflow with planning
        planner = WorkflowPlanner()
        plan = await planner.create_plan(request.description, request.context)

        workflow = Workflow(
            name=request.name,
            description=request.description,
            plan=plan,
            cells=plan.cells,
            execution_mode="cell_based",
            status="ready",
            context=request.context
        )

        workflow_executor.store_workflow(workflow)

        return WorkflowResponse(
            id=workflow.id,
            name=workflow.name,
            description=workflow.description,
            status=workflow.status,
            plan=plan,  # New field
            cells=[cell_to_dict(c) for c in workflow.cells],  # New field
            # generated_code omitted for cell-based workflows
        )
    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))
```

## 15. Success Criteria

1. **User sees step-by-step progress**: Each cell's output displays as it completes
2. **Partial results on failure**: If cell 2/3 fails, cell 1 output is visible
3. **True real-time streaming**: No retrospective progress playback
4. **Cell isolation**: Each cell runs independently with defined inputs/outputs
5. **Debuggability**: Can inspect each cell's input context and output
