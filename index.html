4. Deploy your workflow

Choose your deployment method based on your infrastructure and requirements. Click an option to expand.

🐳 Option A: Docker Recommended ▼

Deploy your generated workflow with Docker for a complete, production-ready solution.

  1. Prerequisites: Install Docker Desktop
  2. Download Workflow Package: After generating your workflow in Step 3, click the "Download Workflow Package" button to get a ZIP file
  3. Extract the Package: Unzip the downloaded file to your desired location (e.g., C:\workflows\my-workflow\)
  4. Configure Environment: In the extracted folder, you'll find a .env.example file. Either:
    • Option 1: Rename .env.example to .env and edit it with your Paradigm API key
    • Option 2: Create a new .env file with: 
      LIGHTON_API_KEY=your_paradigm_api_key_here
LIGHTON_BASE_URL=https://paradigm.lighton.ai
  5. Start with Docker Compose: In the extracted folder, run: 
    docker compose up -d
  6. Access Your Workflow API: Your workflow is now accessible at http://localhost:8000/workflow
✅ Benefits:
  • Complete deployment with FastAPI server included
  • No complex configuration needed - just download, unzip, configure .env, and run
  • Includes API endpoints with automatic documentation at http://localhost:8000/docs
  • Easy to scale and deploy on any Docker-compatible infrastructure (cloud VPS, on-premises, etc.)
☁️ Option B: Vercel Serverless ▼

Deploy your workflow as a serverless function on Vercel for automatic scaling and global distribution.

  1. Create a new directory for your API: Create a new folder on your computer for this deployment
  2. Install Vercel CLI: If you haven't already, install Vercel CLI by running: npm install -g vercel

    def do_POST(self):
        content_length = int(self.headers['Content-Length'])
        post_data = self.rfile.read(content_length)
        data = json.loads(post_data.decode('utf-8'))
        
        # Execute your workflow here
        result = execute_workflow(data.get('query', ''))
        
        self.send_response(200)
        self.send_header('Content-type', 'application/json')
        self.send_header('Access-Control-Allow-Origin', '*')

  3. Set environment variables: In the Vercel dashboard, go to your project settings → Environment Variables and add your Paradigm API key

app = FastAPI()

class WorkflowRequest(BaseModel):
    query: str
    user_id: Optional[str] = None

@app.post("/workflow")
async def execute_workflow_endpoint(request: WorkflowRequest):
    try:
        result = execute_workflow(request.query)
        return {"result": result, "status": "success"}
    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))

if __name__ == "__main__":
    uvicorn.run(app, host="0.0.0.0", port=8000)

5. Integrate your workflow with Paradigm using MCP

🔧 MCP Integration Setup ▼

Connect your generated workflow to Paradigm as an MCP (Model Context Protocol) server, allowing Paradigm agents to query it as an external tool.

ℹ️ What is MCP?

MCP (Model Context Protocol) servers provide external knowledge, data, or tools that Paradigm agents can query. This enables connecting Paradigm with custom systems or third-party providers. Note that responses from an MCP server are not integrated into Paradigm's database - Paradigm depends on the MCP's own search engine, retrieval mechanism, or service quality.

⚠️ Alpha Feature: MCP server integration is currently in Alpha with Sunny Seal (agents required) and will transition to Beta with Tender Tiger. Only System Admins can add or edit MCP servers.

Configuration Steps

Step 1: Prepare Your MCP Server

  1. Download MCP Package: After generating your workflow, click the "Download MCP Package" button to get a ZIP file containing the MCP server
  2. Deploy Your MCP Server: Deploy the MCP server to a publicly accessible URL:
    • For Docker: Deploy to a server and ensure it's accessible (e.g., https://your-server.com:8000)
    • For cloud hosting: Use services like Railway, Render, or your preferred hosting provider
    • For development: Use a tunneling service like ngrok to expose your local server
  3. Note Your MCP Endpoint: Your MCP server will be accessible at a URL like https://your-server.com:8000 (adjust based on your deployment)

Step 2: Register the MCP Server in Paradigm

🔑 Admin Access Required: Only System Admins can configure MCP servers.
  1. Navigate to MCP Settings: In Paradigm, go to Admin > MCP Servers
  2. Create New MCP Server: Click New to register your MCP server
  3. Fill in Registration Details:
    • Name: A human-readable identifier for your workflow (e.g., document-analysis-workflow)
    • URL: The MCP server endpoint URL (e.g., https://your-server.com:8000)
    • Bearer Token: (Optional) Add authentication credentials if your MCP server requires them. The token will be stored encrypted.
    • Enabled: Toggle this to activate the MCP server
  4. Save Configuration: Click save to register the MCP server. At this point, the server is registered but not yet associated with any chat settings.

Step 3: Enable MCP Server in Chat Settings

  1. Navigate to Chat Settings: Go to Chat Settings in Paradigm admin
  2. Select Configuration: Choose the chat configuration where you want to use the MCP server
  3. Enable Agent Mode: Make sure Agent mode is enabled for this chat configuration
  4. Add MCP Server to Agent Tools: Under Agent Tools, add your registered MCP server to the tools list
  5. Save Settings: Save the chat configuration
✅ Ready to Use: Your MCP server is now available to Paradigm agents during chats. The agent will automatically query your workflow when relevant to the user's request.

Example MCP Servers

LightOn provides several pre-built example MCP servers you can explore:

  • Arxiv MCP (https://arxiv-mcp.replit.app) - Query scientific publications
  • Wikipedia MCP (https://wiki-mcp.replit.app) - General knowledge retrieval
  • GitHub Issues MCP (https://github-issues-mcp.replit.app) - Repository issue queries
  • Weather MCP (https://weather-mcp.replit.app) - Real-time weather data
  • File System MCP (https://fs-mcp.replit.app) - Access local/cloud files
💡 Pro Tip: Make sure your workflow MCP server has clear tool descriptions and parameter definitions. Paradigm agents use these descriptions to understand when and how to use your workflow effectively.
